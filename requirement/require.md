# 素珍管理系统 - 需求文档

## 项目背景

构建一个基于 Python 的公司综合管理系统，用于管理公司的租赁、知识产权、合同、专利、车险续期等服务。系统采用全栈 Web 架构，支持 Docker 容器化一键部署。

### 10. 数据导出
- 三种导出方式：导出所有数据 / 按创建时间范围导出 / 按到期时间范围导出
- 支持 JSON 和 Excel (.xlsx) 两种格式（前端默认使用 xlsx）
- Excel 导出使用 openpyxl 生成多 Sheet 工作簿（合同数据 / 专利数据 / 车险数据）
- 表头为中文列名，蓝底白字加粗居中，自动列宽（最大 40 字符），细边框
- 特殊字段翻译：邮件提醒（1/0 → 是/否）、来源（scan/manual → 扫描导入/手动录入）
- 导出文件保存到 `/data/download/`（Docker）或 `data/download/`（本地开发），目录不存在时自动创建
- 文件名格式：`export_YYYY-MM-DD_HH-mm-ss.xlsx`（北京时间戳）
- 前端交互流程：
  1. 用户选择导出方式（全部/按创建时间/按到期时间），选择时间范围时需填写起止日期
  2. 点击"开始导出"→ 弹窗显示"⏳ 正在导出数据..."（无关闭按钮，防止用户中断）
  3. 后端完成查询和文件生成后返回结果
  4. 弹窗自动切换为"✅ 导出完成"，展示各模块导出条数 + 宿主机 Windows 完整文件路径
  5. 用户点击"确定"按钮确认后关闭弹窗
- 路径翻译：`_host_path()` 将容器内 `/data/download/xxx.xlsx` 转为宿主机 `HOST_DATA_DIR\download\xxx.xlsx` 用于前端展示

### 11. 目录扫描 & 批量导入
- 扫描指定目录下的文件，识别哪些文件尚未录入系统
- 支持合同、专利、车险三种模块
- **自动扫描根目录**：打开同步弹窗自动扫描 `/data`，无需用户手动输入路径
- **逐级浏览子目录**：非递归扫描，显示当前目录的子目录列表和未导入文件数，用户可点击进入下级
- **导航控制**：🏠 返回根目录 / ⬆ 返回上级，面包屑显示当前路径
- **目录树展示**：有未导入文件的目录绿色高亮显示数量，已全部导入的目录折叠在底部
- **多选导入**：勾选一个或多个目录，点击"📥 导入选中目录"一键递归导入目录下所有新文件
- **双重去重匹配**：同时用 `file_path`（完整路径）和 `file_name`（原始文件名）判定文件是否已存在，任一匹配即跳过，防止文件移动路径后重复导入
- **源文件名保存**：导入时同时保存 `file_path`（容器内路径）和 `file_name`（原始文件名），用于后续匹配
- 配置通过 `HOST_DATA_DIR` 环境变量传入，后端自动将 Windows 路径翻译为容器内 Linux 路径
- 前端通过 `ScanHelper` 共享模块统一管理，合同/专利/车险三个页面共用

## 核心设计原则

1. **日志管理**：所有操作必须有日志记录（操作时间、操作类型、操作人、详情）
2. **错误处理**：前后端都需有明确的错误处理机制，前端展示友好的错误提示
3. **配置管理**：所有参数从配置文件读取，支持环境变量覆盖
4. **数据字段规范**：所有录入数据必须包含 `创建时间` 和 `到期时间` 字段
5. **到期提醒**：通过 APScheduler 后台调度器每 60 秒检查一次，时间匹配使用 `>=`（非精确等于）避免漂移漏发，`_last_sent_date` 全局变量保证每天只发一次；通过 `scheduler.lock` 文件锁防止 gunicorn 多 worker 重复启动；每条记录可单独设置邮件提醒开关，关闭后该条记录不参与到期提醒；系统配置页面修改提醒时间后自动调用 `reset_send_state()` 重置发送状态
6. **文件类别**：固定可选值为「合同」「专利」「车险续期」
7. **重要等级**：每条业务记录（合同/专利/车险）可设置重要等级：重要、普通、不重要（默认普通），以彩色标签展示
8. **批量操作**：合同/专利/车险页面支持全选、批量设置邮件提醒开关、批量设置重要等级
9. **可配置列显示**：合同/专利/车险页面通过"⚙️ 列显示"按钮勾选要显示的列，配置保存到 localStorage 持久化
10. **权限控制**：系统管理员拥有所有权限；业务员/财务不拥有删除权限、系统配置修改权限、用户管理权限；业务员/财务只能看到同角色用户和自己的日志
11. **时区统一**：SQLite 的 `CURRENT_TIMESTAMP` 返回 UTC 时间，系统通过自定义 SQLite 函数 `beijing_now()` 确保所有时间写入均为北京时间 (UTC+8)
12. **环境识别**：前端通过 `API_BASE === '/api'` 判断是否运行在 Docker 环境，用于同步模态框路径提示切换、IP 获取等场景
13. **IP 获取优先级**：安装脚本获取本机 IP 时优先匹配 WLAN 无线网卡，排除 VMware/VirtualBox 虚拟网卡，避免拿到不可用的 IP
14. **源文件名存储**：每条业务记录存储 `file_name` 字段记录原始文件名，用于扫描去重匹配；新增/编辑页面不提供附件上传控件
15. **宿主机路径展示**：API 响应中包含 `hostFilePath` 字段（Windows 格式路径），前端表格和详情弹窗均展示宿主机路径而非容器内路径
16. **本地文件同步**：通过 `ScanHelper` 前端共享模块 + `POST /api/scan/import-dirs` 后端接口实现，双重去重（file_path + file_name），支持目录树浏览和多选批量导入
17. **信息完整性**：contracts/patents/insurances 三表新增 `is_complete` 字段（INTEGER DEFAULT 1），扫描导入写入 0，手动录入写入 1；`_row_to_dict()` 中动态计算：source='scan' 且必填字段有空值则 `isComplete=false`；前端显示"完整性"列（✅ 完整 / ⚠️ 不完整）
18. **表头排序**：合同/专利/车险页面支持按到期时间、名称、重要程度点击表头升序/降序排列；后端使用列名白名单防 SQL 注入，priority 排序用 CASE WHEN 映射（重要=0 普通=1 不重要=2）；前端排序图标：未选中 ⇅ 灰色，升序 ▲ 蓝色，降序 ▼ 蓝色

## Docker 部署架构

```
用户 → http://localhost:8080
         │
    [nginx:80]  (frontend 容器)
         │
         ├── /            → 静态文件 (frontend/)
         ├── /api/*       → proxy_pass → backend:5000
         │
    [backend:5000]  (Flask 容器, python slim + gunicorn)
         │
         ├── SQLite DB  → /data/suzhen.db  (bind mount 持久化)
         └── Uploads    → /data/uploads    (bind mount 持久化)
```

## 部署方式

### 方式一：一键安装 (setup.bat)

双击项目根目录下的 `setup.bat`，按提示操作：
1. 检测 Docker 环境（未安装则引导下载）
2. 配置国内镜像加速
3. 选择数据存储目录
4. 自动获取本机IP并启动服务

### 方式二：本地开发

```bash
# 使用 .venv 虚拟环境
.venv\Scripts\python.exe backend\run.py
# 访问: http://localhost:5000
```

### 方式三：手动 Docker

```bash
docker compose -f docker/docker-compose.yaml up -d --build
```

### 方式四：客户交付（预构建镜像）

客户无需源码，通过预构建的 Docker 镜像一键启动：

1. 安装 Docker Desktop
2. 解压 `suzhen-system-v1.0.zip`
3. 双击 `setup.bat`，输入端口 → 自动导入镜像并启动
4. 浏览器自动打开，开始使用
5. 双击 `stop.bat` 停止服务（数据通过 Docker 命名卷持久化，不丢失）

开发者构建交付包：运行 `build.bat` → 自动配置 DaoCloud 镜像加速 → 构建镜像 → 导出 tar → 打包 zip。

## 权限分配实现

### 角色定义

| 角色 | 权限范围 |

| `DELETE /api/files/<id>` | `@login_required` → `@admin_required` |
| `DELETE /api/logs` | `@login_required` → `@admin_required` |
| `GET /api/users` | 非管理员追加 `WHERE role = ?` 过滤同角色用户 |
| `GET /api/logs` | 非管理员追加 `WHERE user_id = ?` 过滤自己的日志 |

### 前端实现

- `app.js` 新增 `isAdmin()` 函数和 `applyPermissions()` 函数
- `applyPermissions()` 在 `DOMContentLoaded` 中自动执行，将所有 `.admin-only` 元素 `display: none`
- 各页面的删除按钮、新增/编辑/重置密码按钮、保存配置按钮、清空日志按钮均标记 `admin-only` class
- 导航栏/侧边栏中"用户管理"和"系统配置"链接标记 `admin-only` class
- `users.html`：非管理员访问时自动跳转到仪表盘
- `settings.html`：非管理员访问时所有表单字段设为 `disabled`（只读）

## 可配置列显示实现

合同管理、专利管理、车险续期页面支持自定义列显示：

- 表格上方提供"⚙️ 列显示"按钮，点击弹出勾选面板
- 每列通过 `data-col` 属性关联配置 key
- 默认只勾选：名称、类型、公司、业务员、到期时间、状态、备注
- 配置通过 `localStorage` 持久化（key: `contractColumns` / `patentColumns` / `insuranceColumns`）
- 隐藏列使用 CSS class `.col-hidden { display: none !important; }`
- 空数据行 `colspan` 动态计算可见列数
